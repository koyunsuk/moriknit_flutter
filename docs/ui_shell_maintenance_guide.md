# UI Shell Maintenance Guide

## 목표

- 화면이 늘어나도 톤이 흔들리지 않도록 공통 껍데기를 유지한다.

## 공통 구조

### 1. Header

- 공통 헤더 컴포넌트를 사용한다.
- 제목, 부제목, 칩, 보더, 배경 알파는 공통 규칙을 따른다.

### 2. Body

- 실제 데이터와 액션이 들어가는 영역
- 공통 배경은 메인 쉘에서 한 번만 처리한다.

### 3. Footer

- dock 또는 하단 고정 액션
- 화면 구조를 잡아주는 역할만 한다.

## 공통 컴포넌트 우선순위

- 헤더
- 카드
- empty / loading / error
- section title
- form field
- CTA button
- chip / badge

## 유지보수 규칙

- 같은 역할의 UI는 같은 컴포넌트를 쓴다.
- 화면마다 비슷한 카드나 제목 스타일을 새로 만들지 않는다.
- subtitle은 기능 우선순위처럼 보이지 않도록 하되, 테마감은 유지한다.
- 상세/입력 화면과 메인 탭 화면의 헤더 노출 규칙을 분리한다.
